Relatórios agregados por URLs permitem ao administrador do sítio agrupar livremente URLs disponibilizando uma forma fácil de gerar-se relatórios de acesso por área.

Requisitos

O módulo central Statistics é a opção registro de acessos na página Configurações de log de acesso precisam estar habilitados para que se possa gerar relatórios de acesso agrupados por URLs.

Ao habilitar o registro de acessos é importante lembrar que esses registros devem ser preservados por tanto tempo quanto se queira produzir relatórios agregados por URLs.

Módulos recomendados

Além dos dois requisitos obrigatórios mencionados acima há dois módulos recomendados:

  • o módulo Statistics Advanced corrige bugs no módulo central Statistics
  • o módulo Browscap permite que o módulo Statistics Advanced separe tráfego normal do tráfigo gerado por robôs

Uso

Para gerar relatórios de acesso agregados por URLs vocÊ precisa antes criar ao menos um agregador de URLs. Isso pode ser feito na página Criar agregador de URLs.

A lista de URLs dos agregadores aceita URLs com padrões.

Com ao menos um agregador criado você pode criar um relatório na página de relatório Agregado por URLs.

Esta página também aceita um período definido no fim da URL. Os períodos suportados são:

  • today - Hoje
  • yesterday - Ontem
  • week_current - Semana atual
  • week_last - Semana passada
  • week_last2 - Últimas 2 semanas
  • month_current - Mês atual
  • month_last - Mês passado
  • month_last3 - Últimos 3 meses
  • month_last6 - Últimos 6 meses
  • quarter_current - Trimestre atual
  • quarter_last - Último trimestre
  • year_current - Ano atual
  • year_last - Último ano
  • custom_days - Número personalizado de dias

    Para cusom_days você pode definir um parâmetro extra indicando o número de dias a serem cobertos pelo relatório.

Exemplos:

Para obter um relatório cobrindo o mês atual:

admin/reports/statspro/path_aggregated/month_current

Para obter um relatório cobrindo os últimos 3 meses:

admin/reports/statspro/path_aggregated/month_last3

Para obter um relatório cobrindo os últimos 20 dias:

admin/reports/statspro/path_aggregated/custom_days/20

Observações

É importante notar que a geração de relatórios agregados por URLs pode ser um processo pesado especialmente se houver muitos acessos a serem analisados, seja porque o período coberto pelo relatório é muito grande, seja porque o sítio tem muitos acessos registrados.
