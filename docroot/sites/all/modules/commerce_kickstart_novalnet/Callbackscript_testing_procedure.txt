CALLBACK SCRIPT TESTING PROCEDURE
---------------------------------

    Please follow the below mentioned steps to call the callback script for all the payment types.

    Url: <siteurl>/<callback script file>

    Ex: https://commercekickstart.novalnet.de/callback_novalnet

Note:
-----
(kindly make the below changes in configuration, Login as ADMIN and goto store settings-> Novalnet payments-> Merchant script management)
------------------------------------------------------------------------------------------------------------------------------

  1) Enable test mode  = Yes;

  2) Enable debug mode = Yes;

  3) a) formatted URL for Invoice and Prepayment:-

       <Site URL>/callback_novalnet?vendor_id=4&status=100&payment_type=INVOICE_CREDIT&tid_payment=12675800001204435&amount=3778&tid=12675800001204435

  Payment type:
  -------------

  INVOICE_CREDIT (Invoice and Prepayment)


    b) formatted url for other payment types which are mentioned below: (except Invoice / Prepayment):-

  Payment types:
  --------------

  PAYPAL (PayPal)
  ONLINE_TRANSFER (Instant Bank Transfer)
  CREDITCARD (Credit Card and Credit Card 3D Secure)
  IDEAL (iDEAL)
  DIRECT_DEBIT_SEPA (Direct Debit SEPA)

       <Site URL>/callback_novalnet?vendor_id=4&status=100&payment_type=PAYPAL&amount=3778&tid=12675800001204435

  Please use the respective payment type in the param "payment_type". (EX : payment_type=PAYPAL)


  Parameters:
  -----------

    vendor_id          - Merchant ID

    status             - Successful payment transaction value

    payment_type       - Type of payment process

    tid_payment        - Existing appropriate order transaction id ( only for Invoice and Prepayment payment methods )

    amount             - Customer paid amount in cents

    tid                - Callback script Transaction TID
